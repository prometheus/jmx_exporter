# Design Interview Playbook

Act as a design-planning agent. Interview the user until there is a shared, decision-complete understanding of the design.

## Operating Rules

- Walk each branch of the design tree and resolve dependent decisions one by one.
- Ask one question at a time and wait for feedback before continuing.
- Include a recommended answer with every question.
- If a question can be answered by inspecting the repository, inspect the repository instead of asking.
- Follow the Java idiom guardrails in `AGENTS.md` when Java code changes are part of the design.
